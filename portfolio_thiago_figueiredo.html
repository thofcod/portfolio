Experiências e Cases Relevantes

Case 1: Implantação Philips Tasy – Migração de Dados de Sistema Legado (Unimed)

Contexto:
Projeto de implantação do Philips Tasy envolvendo a migração de grandes volumes de dados provenientes de sistemas legados, com diferenças estruturais, regras de negócio distintas e necessidade de alta confiabilidade para início da operação.

Atuação:

  • Participação ativa e contínua nas frentes de migração de dados durante o projeto de implantação do Tasy.
  • Participação técnica no mapeamento DE-PARA entre estruturas do sistema legado e o modelo de dados do Tasy.
  • Análise das tabelas, chaves e relacionamentos envolvidos nos processos de migração.
  • Desenvolvimento de scripts SQL e PL/SQL para:
    • saneamento de dados inconsistentes,
    • padronização de formatos,
    • validação de integridade referencial antes da carga.
  • Desenvolvimento e Preparação das cargas, execução controlada em ambientes de teste e homologação e conferência dos dados pós-importação.
  • Suporte técnico durante ciclos de teste e na fase de virada de sistema (go-live), atuando na identificação e correção rápida de inconsistências.

Impacto:
Contribuição direta para a realização das cargas de dados de forma consistente e segura, reduzindo riscos na virada do sistema e garantindo a disponibilidade de informações confiáveis no Tasy desde o início da operação.

Case 2: Automação e Envio de Eventos do eSocial após Virada para o Tasy

Contexto:
Após a virada do sistema para o Philips Tasy, a geração e o envio dos eventos do eSocial (S-1200, S-1210 e S-2300) deixaram de ser realizados devido a limitações na integração nativa entre Tasy e Senior. A situação gerou meses de atraso no envio dos eventos, com impacto regulatório e dependência de orçamentos elevados para customização da integração oficial.

Atuação:

  • Engenharia de Dados (Oracle / PL/SQL):
    • Desenvolvimento de procedures e scripts SQL para extração dos dados diretamente das tabelas do Tasy (folha, prestadores e cooperados).
    • Tratamento, padronização e formatação dos dados conforme os layouts oficiais do eSocial.
    • Implementação de validações de obrigatoriedade e consistência antes da geração dos eventos.
  • Solução de Integração Customizada:
    • Criação de uma aplicação em Python para consumir as procedures, orquestrar os dados e gerar os XMLs do eSocial.
    • Desenvolvimento de interface em React para permitir:
      • acompanhamento dos envios,
      • visualização e tratamento de erros,
      • validação de dados obrigatórios,
      • geração e download de arquivos por lote.
  • Conformidade e Controle:
    • Validação estrutural dos XMLs (XSD).
    • Implementação de mecanismos para rastreabilidade dos eventos enviados, rejeitados ou pendentes.

Impacto:
Restabelecimento do envio do eSocial após a virada para o Tasy, eliminação do processo manual, redução do risco regulatório e criação de uma solução interna funcional enquanto a integração oficial não era viável.

Case 3: Automação do Fluxo de Requisições OPME no Tasy para Ordens de Serviços

Contexto:
Processo manual e fragmentado para tratamento de requisições de OPME, gerando retrabalho e atrasos na criação de Ordens de Serviço no sistema.

Atuação:

  • Apoio técnico na definição do fluxo de dados do processo.
  • Desenvolvimento de rotinas em PL/SQL para automatizar etapas do pipeline.
  • Automatização do cadastro das requisições e geração de OS conforme regras pré-definidas.
  • Validação técnica dos resultados junto às áreas envolvidas.

Impacto:
Redução de atividades manuais, maior padronização do processo e melhoria no tempo de resposta operacional.

Case 4: Desenvolvimento de Relatórios Operacionais e Gerenciais Avançados no Tasy

Contexto:
Necessidade contínua de relatórios confiáveis e performáticos para suportar as áreas assistencial, auditoria, financeira e gestão médica, envolvendo regras de negócio complexas e grandes volumes de dados no Tasy.

Atuação:

  • Desenvolvimento de queries SQL complexas para extração e consolidação de dados a partir de múltiplas tabelas do Tasy.
  • Criação de procedures em PL/SQL para aplicação de regras de negócio, cálculos e tratamentos necessários à geração dos relatórios.
  • Estruturação de packages para centralizar regras, padronizar consultas e facilitar manutenção e evolução dos relatórios.
  • Implementação de jobs automatizados para execução recorrente, geração de dados consolidados e suporte a relatórios dinâmicos e parametrizáveis.
  • Validação técnica dos resultados junto às áreas solicitantes, com ajustes conforme regras operacionais e assistenciais.

Impacto:
Relatórios mais confiáveis, performáticos e alinhados às regras de negócio, redução de extrações manuais e maior autonomia das áreas no acesso às informações.

Case 5: Documentação Técnica e Apoio à Comunicação entre Negócio e TI

Contexto:
Demandas evolutivas no Tasy apresentavam divergências de entendimento entre regras de negócio e a forma como os dados eram persistidos e extraídos no banco Oracle, gerando retrabalho em relatórios e ajustes técnicos.

Atuação:

  • Análise direta das tabelas, relacionamentos e campos envolvidos em cada demanda.
  • Criação de queries SQL para validar regras de negócio, identificar inconsistências e exemplificar cenários reais de dados.
  • Documentação objetiva com:
    • tabelas e campos utilizados,
    • regras de cálculo aplicadas,
    • consultas SQL de apoio para conferência e validação.
  • Registro dos scripts e ajustes executados para facilitar reutilização e suporte futuro.

Impacto:
Redução de ambiguidades técnicas, menor retrabalho em ajustes e maior agilidade na validação das entregas.
